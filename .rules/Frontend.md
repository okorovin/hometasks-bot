- для страниц и вложенных страниц используем подход как в примерах из папки @.example
- создаем все нужные куски pages, features, shared, entities
- для отдельных компонент делаем свои файлы в нужных срезах
- все ссылки должны быть рабочими. если надо делаем просто моковые страницы
- создавай сторы как в примерах. можно делать моковые функции
- при генерации типов, старайся использовать interface вместо type где можно
- старайся не писать сложные типы инлайново, а выносить в именнованные типы
